# PKO.R01 date header rule policy - CR-DB-0114A

- MB_ID: CR-DB-0114A-DATE-HEADERS-TRACE-DB-PATCH-AND-PKO-REGEN
- Status: ACTIVE

## Rule
Date headers in main.xlsx > F.D.KUM.01!E7:P7 and R7:AC7 must be real Excel dates generated by formulas, not visible code text.

## Correct formula pattern
- January 2026 month-end is DATE(2026,2,0).
- DATE(2026,1,0) returns 2025-12-31 and is not January 2026.

## DB policy
The canonical date header rules are registered in pko_date_header_rule_registry and traced in pko_date_header_migration_audit.

## Output policy
Current WBS short outputs are generated under 04.PERIODS\\2026-05_LOCKED.V1\\PKO\\R01\\current.
